# 提交规则

> 最后更新: 2026-04-22
> 状态: 当前有效提交治理规则

## 1. 目的

统一仓库的提交信息格式。

## 2. commit message 格式

推荐格式:

`<type>(<scope>): <description>`

允许的 `type`:

- `feat`: 新功能
- `fix`: 修复
- `docs`: 文档
- `refactor`: 重构
- `test`: 测试
- `chore`: 杂项
- `build`: 构建
- `ci`: CI
- `perf`: 性能
- `style`: 格式
- `revert`: 回退

允许的 `scope`（常用）:

- `entity`: 实体相关命令
- `chapter`: 章节命令
- `consistency`: 一致性校验
- `projection`: 投影管理
- `context`: 上下文
- `outline`: 大纲
- `brainstorm`: 灵感
- `export`: 导出
- `stats`: 统计
- `doctor`: 校验
- `protocol`: 协议层
- `services`: 服务层
- `utils`: 工具层

约束:

- description 简洁明确
- 不使用空泛描述如 "update code" / "fix bug"
- 中文 description 可接受

## 3. 禁止的协作元信息

默认禁止:

- `Co-Authored-By:`
- `Co-authored-by:`
- `Pair-Programmed-By:`
- AI 工具签名或"Generated by ..."类尾注
