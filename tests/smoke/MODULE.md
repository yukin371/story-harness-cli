# tests/smoke 模块说明

> 最后更新: 2026-04-23
> 状态: 当前有效模块文档

## 1. 模块职责

- 维护 CLI 关键链路的 smoke test
- 为样例工程和高风险命令提供端到端回归
- 验证命令输出结构与关键字段，而不是追求所有内部细节覆盖

## 2. Owns

- `tests/smoke/test_*.py`

## 3. 不变量

- 测试应优先通过 `cli.main()` 调用真实命令入口
- 样例工程 smoke test 应复制样例到临时目录后再运行
- 新样例工程落地后，若作为回归基线，应补对应 smoke test

## 4. 文档同步触发条件

- 新增样例工程 smoke test
- 关键命令链路变化
- smoke 测试约定变化
